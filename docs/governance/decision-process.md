# Processo de decisao

Status: ACCEPTED

## Quando criar ADR

Crie ADR para decisoes que alterem arquitetura, runtime, CI, seguranca, persistencia, fronteiras de
modulo ou direcao irreversivel do produto.

## Quando criar RFC

Crie RFC para propostas ainda abertas, com alternativas e impactos. Uma RFC aceita pode gerar ADR.

## Decisoes pendentes

Decisoes de dominio ainda nao tomadas devem ser registradas em
[decisoes de dominio pendentes](../domain/open-domain-decisions.md).

## Regra de substituicao

Nao edite uma decisao historica para mudar seu significado. Crie um novo ADR e marque a relacao.
